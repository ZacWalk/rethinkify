  • diffractor app

  • nnc

    For a while I was a compiler engineer, and I always had this idea: if you could just compile an LLM to run on the CPU, then Jensen Huang would be out of business. NNC is my attempt — an LLM compiler and JIT. It runs Gemma 3 at about 200 ms per token on my PC with AVX2. Jensen is safe for now, but I am excited about FP8 landing in CPU SIMD. Inference is mostly limited by memory bandwidth, so FP8 would let the model run a lot more efficiently. Intel has already committed to FP8 vector instructions in AVX10.2, and hopefully AMD follows with FP8 SIMD in Zen 6 or 7. Once that lands, a CPU-only LLM stack starts to look a lot more interesting.

    github.com/ZacWalk/nnc

  • spike-db

    A time-series database built for financial market data. I use it in my trading experiments and in Alpha Galactica. Its one trick is very fast in-order iteration of market data — handy if you ever need that sort of thing.

    github.com/ZacWalk/spike-db

  • dos4z

    A while back, Microsoft open-sourced MS-DOS 4.0 (with a little help from Scott Hanselman). I wanted to add large hard drive support so my IBM 5150 could use bigger disks. The tricky part is that building DOS relies on some bundled DOS-only compilers and assemblers — so I wrote my own DOS shim to run those old programs on a modern Windows machine. You can now build this version of DOS on a modern PC, and my IBM 5150 boots from the result. I still have not figured out the large disk support though.

    github.com/ZacWalk/dos4z

  • electron-bench

    Measures the cost of sending messages across Electron's communication paths. I wrote it to improve performance while I was working on Microsoft Teams. Teams does not use Electron any more, but if you do, this might help.

    github.com/ZacWalk/electron-bench

  • rust-http-sys

    A few years ago I started using Rust at Microsoft to improve performance for some of the services that make Office work. They serve hundreds of millions of people, so making them faster adds up to a lot of money saved. One of my first tasks was to prototype this wrapper for http.sys, the Windows kernel-mode web server. I was quite proud of it — until one of our Rust experts told me it looked like a C++ developer trying to write Rust. Fair enough: I am a C++ guy trying to write Rust :)

    github.com/ZacWalk/rust-http-sys

  • stuntcarracer

    I loved Stunt Car Racer as a kid on my Commodore 64. The code from the Amiga version was floating around, so I grabbed a copy and rewrote it to use floating-point math instead of fixed-point. The fact that the original 3D graphics were done entirely in fixed-point shows how amazing those old-school programmers were. On modern hardware floating point costs nothing, but I did introduce a bug where you occasionally fall through the track. It has me stumped — even Claude Opus could not find the root cause. Maybe I will switch it all back to fixed point if I get the time.

    github.com/ZacWalk/stuntcarracer

  • low-ban

    Around 2015 I built low-ban as a hackathon project at Microsoft, while I was working on Skype. The idea was a video call that could run on a 2G network, at around 10 kbps. At that speed all you can send is a blurry mess, so instead I tracked facial landmarks and sent just the coordinates of someone's expression. That way you could at least see whether they were smiling. There was some interest — I even demoed it to the head of design for Skype, who wore all black and really looked the part. In the end nothing came of it, and now we all have plenty of bandwidth for high-resolution video.

    github.com/ZacWalk/low-ban

  • system-stuff

    Way back around 2000, the company I worked for still supported Windows 95 and 98 for the desktop app we sold. I usually volunteered to test on those old platforms, and I had written my own task-manager-style program. The Windows 95 task manager was very basic, and I decided I could do better. Every five years or so I get it building again and make a few tweaks. It started life as sys-mate and is now called, more generically, system-stuff.

    github.com/ZacWalk/system-stuff

  • noterad

    github.com/ZacWalk/noterad

  • ergo-active

    Back in the late 90s my London neighbour, a Swedish furniture designer, came up with the idea for Ergo Active over beers. The premise was simple: remind you to take a break when you have been at the computer too long. We had so much fun making it, and filled it with advice that all started with “experts recommend…” Once it was finished we had no idea what to do with it — there was no app store back then. Luckily it is preserved here. If you forget to take breaks, this is the app for you.

    github.com/ZacWalk/ergo-active

  • nn

    Fashion-MNIST is a dataset of pictures of clothes, used as a machine learning challenge: train a model to tell the items apart. As a learning exercise, I wrote my own multi-layer perceptron to see how deep learning actually works under the hood. It does about as well as any other multi-layer perceptron. I tuned it for modern SIMD hardware, so it is fast — for a CPU. If you ever need to tell a sock from a shirt, this is the program!

    github.com/ZacWalk/nn

  • potato

    Back in the 2000s I moved off C++ to work on a web project for a while. I wanted to learn CSS, and decided the best way was to write my own web browser in C++. Here is the result — probably the worst web browser in the world. I try to keep it working well enough to show Wikipedia. Right now it is broken because Wikipedia keeps adopting more modern CSS. It still works on Wikipedia from 2015 — so very handy if you ever travel back in time.
